# Test Report v1

对 `xuanji-decision` 做第二轮手工压测，目标：

1. 看是否能稳定处理 Agent 项目中的真实分叉点
2. 看输出是否兼具“像”与“可执行”
3. 找出容易穿帮、容易拒断或需要补规则的场景

---

## 总结

### 当前结论

- 适合：产品路线、商业模式、执行节奏、内容策略、用户定位这类“项目分叉点”
- 勉强适合：品牌调性、表达风格、视觉气质这类偏审美问题
- 不适合：明显有客观答案的选择、信息严重不足的空问题、无真实项目上下文的情绪摇摆

### 当前阶段判断

skill 已经进入 **可打样使用阶段**，但还没到“无脑即插即用”。

最稳的场景：
- 先做 A 还是先做 B
- 先验证还是先包装
- 先卖服务还是先做产品
- 先做单点还是先铺平台

最容易漂的场景：
- 文案要高级一点还是亲切一点
- logo 要不要更神秘一点
- 页面主色用黑金还是奶白

## Case 1：先做内容验证，还是先做 MVP

### 输入
- 项目：一个新 AI 产品
- 分叉点：先做内容验证还是先做 MVP
- A：先做内容验证，看用户是否真买单
- B：先做 MVP，先把产品做出来
- A 成立：验证需求快、成本低、能先拿反馈
- B 成立：没有产品就没有真实体验，内容容易空
- 卡点：两边都合理，迟迟拍不了板
- 截止：本周内必须定

### 预期
应优先断为“看当前最缺什么”，而不是泛泛说都可以。

### 实测判断
- 滴天髓：看当前势是否已经在外部表达侧形成
- 穷通宝鉴：看当前最缺验证还是最缺交付物
- 渊海子平：看主格是“先成点”还是“先闭环”
- 三命通会：看此时是否宜先起势再承接

### 结果
**通过，且很稳。**

### 原因
这是四书体系最容易咬住的场景：
- 有明确分叉
- 两边都成立
- 不是审美题
- 项目推进会被真实阻塞

### 优化建议
无须大改，只需继续补内容/产品类专门词库。

## Case 2：做工具型产品，还是做情绪陪伴型产品

### 输入
- 项目：AI 陪伴类方向探索
- 分叉点：工具型 vs 情绪型
- A：强调效率、功能、决策辅助
- B：强调陪伴、感受、关系感
- A 成立：更清晰、更易定价、更像工具
- B 成立：情绪价值更高，更容易被用户持续使用
- 卡点：担心做 A 太干，做 B 太虚
- 截止：本周需要定产品路线

### 预期
应围绕“主格局”和“当前最缺什么”来断，而不是停留在风格偏好。

### 结果
**通过，但需要项目主格输入更明确。**

### 问题
如果没有 `primary_goal`，模型容易泛说：
- A 成局，B 易散神
- 当前最缺聚焦

这些话方向对，但不够咬项目。

### 优化建议
在输入结构中新增：
- `project_stage`
- `primary_goal`

## Case 3：先卖服务，还是先做订阅工具

### 输入
- 项目：个人创业项目
- 分叉点：服务优先 vs 工具优先
- A：先卖服务，先有现金流
- B：先做工具，争取长期可规模化
- A 成立：更快见钱，更易验证
- B 成立：更能做资产，不易被交付绑死
- 卡点：短期生存和长期理想冲突
- 截止：这月内要定

### 结果
**通过，适配度高。**

### 原因
这类场景天然适合：
- 穷通宝鉴：眼下最缺现金流还是资产性
- 渊海子平：哪个更成局
- 三命通会：是否应先以 A 养 B

### 当前最佳断法
不是“永远选服务”或“永远选工具”，而是：
- 先取 A 补命
- 后纳 B 成局

### 优化建议
给商业模式场景新增更多“近远、养局、补命、活血、生肌”类词库。

## Case 4：品牌调性做高冷专业，还是温柔陪伴

### 输入
- 项目：AI 决策助手品牌定位
- 分叉点：高冷专业 vs 温柔陪伴
- A：高冷、神秘、专业
- B：温柔、有人味、可亲近
- A 成立：更有权威感，更高级
- B 成立：更容易拉近距离，更有传播感
- 卡点：怕太冷没人靠近，太暖又显得掉价

### 结果
**勉强可断，但不够稳。**

### 原因
这类问题偏审美与传播策略，若缺少：
- 用户画像
- 渠道信息
- 当前目标（信任 / 传播 / 转化）

则玄断很容易漂成漂亮空话。

### 建议
把这类问题列为“弱适用场景”：
- 可断，但应优先要求补充 `primary_goal` 与 `channel`

## Case 5：现在发版本，还是再打磨一周

### 输入
- 项目：已有可运行版本
- 分叉点：今天发 vs 再打一周
- A：今天就发
- B：再磨一周再发
- A 成立：抓窗口、先拿反馈
- B 成立：减少低级错误、避免第一印象太差
- 卡点：怕太快翻车，也怕太慢错过窗口

### 结果
**通过，且非常适合 skill。**

### 原因
这是典型的“时机问题”，最适合《三命通会》。

### 最好用的断法
- 看是否得令
- 看当前最忌过急还是过迟
- 看是该先试一步还是一步到位

### 建议
后续专门补一组“执行节奏”词库。

## Case 6：做女性用户，还是先做泛用户

### 输入
- 项目：AI 陪伴类产品定位
- 分叉点：先做女性用户 vs 泛用户
- A：聚焦女性高情绪价值需求
- B：面向所有用户做更大盘
- A 成立：更容易打透、更易形成鲜明产品感
- B 成立：市场更大、不容易被说太窄
- 卡点：怕做窄了上限低，也怕做广了打不透

### 结果
**通过。**

### 原因
这类问题适合“先聚后扩”的格局判断。

### 推荐断法
- 渊海子平：先成一局，再谈兼容
- 穷通宝鉴：当前最缺第一批铁用户，还是更大注意力

### 建议
继续补“窄 / 广、上 / 下、聚 / 散”表达。

## Case 7：让用户选黑金视觉还是奶白视觉

### 输入
- 项目：品牌视觉设计
- 分叉点：黑金 vs 奶白
- 卡点：都挺好看，不知道哪个更高级

### 结果
**建议拒断或弱断。**

### 原因
如果没有：
- 品牌调性
- 用户画像
- 渠道信息
- 首页转化目标

这类问题太空。

### 拒断建议
- 此局未成，不足起断。先补品牌目标与受众信息。
- 眼下缺的不是玄断，而是视觉服务于什么目的。

## Case 8：已有明显客观优解时是否会误断

### 输入
- A 成本 1 万、合规、两周上线
- B 成本 8 万、存在法务风险、三个月上线
- 用户仍想用玄学断一下

### 结果
**必须拒断。**

### 当前状态
skill 文档里写了拒断原则，但还缺更强的拒断模板。

### 建议补充
统一拒断短句：
- 此事不在局中，在账上。
- 胜负过明，不必强起玄断。
- 眼下不缺卜辞，缺的是把账与风险算明。
